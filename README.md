# ✂️ مشروع BarberFlow-Pro

منصة متكاملة لتنظيم حجوزات صالونات الحلاقة وتسهيل وصول الزبائن إليها عبر واجهة عصرية ونظام ربط مباشر.


---


## 📂 هيكلية ملفات المشروع (Project Structure)


تم تنظيم المشروع بشكل مديولار (Modular) لسهولة التطوير:


/
├── images/             # الصور الثابتة وشعار الموقع
├── icons/              # الأيقونات المخصصة (إن وجدت)

├── index.html              # الصفحة الرئيسية للموقع
├── salons.html             # صفحة استعراض الصالونات والبحث
├── login.html              # صفحة دخول أصحاب الصالونات
├── register.html           # صفحة تسجيل صالون جديد
├── dashboard.html          # لوحة تحكم الحلاق
│
├── css/                    # مجلد ملفات التنسيق
│   ├── style.css           # التنسيقات العامة والمتغيرات
│   ├── salons.css          # تنسيقات صفحة العرض والبحث
│   ├── dashboard.css       # تنسيقات لوحة الإدارة
│   └── card.css            # تنسيق مكون بطاقة الصالون
│
└── js/                     # مجلد العمليات البرمجية
    ├── main.js             # التفاعلات العامة للموقع
    ├── auth.js             # دوال تسجيل الدخول والخروج (Logic)
    ├── login.js            # التعامل مع نموذج الدخول
    ├── register.js         # التعامل مع نموذج التسجيل
    ├── salons.js           # جلب البيانات من Firestore وعرضها
    ├── dashboard.js        # إدارة وظائف لوحة التحكم
    ├── salonCard.js        # مكون إنشاء بطاقة الصالون برمجياً
    └── modules/
        └── firebase-init.js # ملف تهيئة وربط Firebase


---


## 🎨 دليل الهوية البرصرية (Styling Guide)


تم استخدام متغيرات CSS في ملف `style.css` لتوحيد الألوان في كامل الموقع:


- `--primary`: #1a1a1a (اللون الأساسي الداكن)


- `--gold`: #d4af37 (اللون الذهبي للأزرار والعناوين)


- `--bg-light`: #f8f9fa (خلفية الصفحات)


- `--wa-green`: #25D366 (لون علامة الواتساب)


- `--error-red`: #ff4d4d (التنبيهات والأخطاء)


---


## 🛠 القواعد البرمجية المتبعة (Coding Standards)


1. **تنسيق الكود**: تم ترك فراغ (سطرين) بين كل سطر برمجي والآخر لسهولة القراءة والتطوير لاحقاً.


2. **التعليقات**: كل كتلة برمجية مزودة بتعليق يوضح وظيفتها باللغة العربية.


3. **قاعدة البيانات**: نستخدم Firebase Firestore لتخزين بيانات الصالونات و Firebase Auth لإدارة الحسابات.

## 🚀 ملاحظات للمستقبل


- عند إضافة أي حقل جديد في قاعدة البيانات، يجب تحديث ملف `salonCard.js` ليتم عرضه في البطاقة.


- الموقع يتطلب تشغيله عبر سيرفر محلي (مثل Live Server في VS Code) لدعم خاصية الـ ES6 Modules في JavaScript.
